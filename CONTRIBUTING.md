# Contributing to ApplyTrail

Thanks for taking the time to contribute. ApplyTrail is a small open-source project built by a job seeker, for job seekers — all skill levels are welcome.

## Ways to contribute

**Found a bug?**
Open an issue and describe what happened, what you expected to happen, and what browser / OS you're on. Screenshots help.

**Have an idea?**
Open an issue with the "enhancement" label. Describe the problem it solves, not just the feature — that helps prioritize.

**Want to submit code?**
1. Fork the repo
2. Create a branch: `git checkout -b your-feature-name`
3. Make your changes
4. Test that the app still runs: `npm run dev`
5. Submit a pull request with a short description of what you changed and why

## Getting set up locally

Follow the [README](README.md) setup guide. The "New to this? Start here" section at the top is written for complete beginners — start there if you're new to running Node.js apps.

## A few guidelines

- Keep pull requests focused — one change per PR is easier to review
- If you're adding a new feature, open an issue first so we can discuss it before you spend time building it
- Code style: TypeScript, Tailwind for styling, follow the patterns already in the codebase

## Questions?

Open an issue or reach out on LinkedIn. DMs are open.
